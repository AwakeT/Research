# ASM建图架构方案设计与优化

> **来源文档**：方案设计文档/ASM建图架构.pdf、ASM架构（JQR）.pdf
> **评审维度**：系统和方案设计
> **更新时间**：2026-04

## 一、设计背景与目标

原MapNav框架采用语义分割方式处理RGB图像，通过Semantic Segmentation生成Masked Objects，存在语义粒度粗、无法识别具体物品实例的问题。ASM（Active Semantic Map）的设计目标是构建一套**实例级语义地图构建系统**，能够在机器人移动过程中实时检测、分割物体并投影到鸟瞰图地图上，支撑VLN找物任务。

后续ASM从研发设备（ZED相机）迁移至JQR量产设备时进行了架构优化，包括障碍地图改用导航处提供的地图、语义信息存储增加DB持久化、新增语义信息动态更新策略。

## 二、整体架构设计

### 2.1 研发版架构

| 层级 | 模块 | 功能 |
|------|------|------|
| 输入层 | 深度信息、机器人位姿、RGB图像 | 传感器数据接入 |
| 语义提取层 | Grounding-DINO检测 → MobileSAM分割 → 语义标签提取 | 目标检测+分割 |
| 鸟瞰投影层 | 障碍地图生成 + agent轨迹 + 目标物坐标投影 | 坐标转换与地图融合 |
| 模态融合层 | 障碍地图+语义信息+轨迹 → 统一ASM地图 | ASM地图输出 |

### 2.2 JQR量产版架构优化

| 层级 | 变更说明 |
|------|---------|
| 语义提取层 | 新增**json信息存储 + DB信息存储**双通道 |
| 鸟瞰投影层 | 障碍地图改为**导航处提供**（替代自建深度投影），降低计算开销 |

**DB存储新增字段**：存在状态（existence_status）、last_show_time（最后可见时间）

## 三、RGB数据处理流程

### 3.1 对象检测与分割

```
RGB图像 → Grounding-DINO检测 → 检测框+语义标签
    → MobileSAM分割 → 对象掩码(mask)
    → 保存：JSON文件 + 标注图像 + 语义数据
```

### 3.2 语义地图更新（ValueMap.add_object）

```
对象信息（像素坐标）+ 相机内参 + 位姿（外参）
    → 2D掩码投影到3D空间 → 转换到地图坐标系
    → 更新语义地图 → 记录对象位置和属性
```

## 四、深度数据处理流程

```
原始深度图像 → 深度值裁剪 → 有效像素比例检查 → 归一化
    → 3D点云 + 相机内参/外参 → 鸟瞰图投影
    → 更新障碍物占用栅格 + 可通行区域
```

**更新频率**：障碍地图10Hz，语义信息2Hz（异步解耦）

## 五、ASM实时更新闭环（时序设计）

系统涉及6个核心模块：ASMSystem、ZEDDataSubscriber、PoseBuffer、ObstacleMap、ValueMap、RealObjectDetector。

**三阶段时序**：
1. **传感器数据到达**：depth_callback() → 深度有效性检查 → 取latest_pose → 生成depth-pose pair
2. **主循环处理**：spin_once() → 移动/旋转阈值检查 → update_map() → 更新占据栅格+轨迹 → [每5步] detect(rgb) → add_object() → 发布occupancy_grid
3. **定期落盘**：每10步 → 保存map.png + performance_stats

## 六、动态地图更新方案

### 6.1 问题现状

当前ASM地图仅追加式更新，未考虑场景变化后地图无法适配后续任务。

### 6.2 三种候选方案

| 方案 | 核心思路 | 优点 | 存在问题 |
|------|---------|------|---------|
| 基于agent半径 | 3m圈主图（长期记忆）+ 1m圈子图（短时记忆），匹配-融合-写回同步 | 区分长期/短期记忆 | 双地图+双向同步工程复杂；位姿漂移时把错误写回主图 |
| 基于场景变化 | 初始可见范围存为"参考帧"，后续做"变化评分"，超阈值整体替换 | 按需更新 | 评价函数设计困难 |
| 基于任务预测 | VLN推理给出目标点，按最短路径+视场锥形选取更新范围 | 与任务强耦合 | 依赖VLN推理质量 |

### 6.3 采用方案：评分驱动的语义信息更新策略

最终设计了**首次观测+持续维护**两阶段策略：

**首次观测**：检测到新目标物 → 更新物品信息、时间戳至json（覆盖式）+ db（保留式）

**持续维护**：
```
物品是否被检测（基于最佳检测距离3m）？
├── 是 → 评分++ → 位置变动>50cm？ → 是：更新坐标 / 否：返回
└── 否 → 评分-- → 评分<阈值？ → 是：删除语义信息 / 否：返回
```

**核心设计要点**：评分机制管理物品生命周期 + 50cm位置变动阈值 + json覆盖式+db保留式双存储策略

## 八、设计挑战与创新点

1. **实例级语义映射**：从传统语义分割升级为Grounding-DINO+MobileSAM的实例级检测分割，支持开放词汇物体识别
2. **多模态异步解耦**：RGB语义提取2Hz与Depth几何投影10Hz异步更新，平衡实时性与计算开销
3. **研发到量产的架构适配**：将自建障碍地图替换为导航处提供的地图，降低算力需求
4. **评分驱动的动态更新**：基于证据计数的语义信息生命周期管理，在首次观测与持续维护中平衡实时性与稳定性
5. **存储效率极致优化**：文件体积缩小3267倍，加载时间提升89倍
