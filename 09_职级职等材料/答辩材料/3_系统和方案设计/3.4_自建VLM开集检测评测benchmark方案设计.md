# 自建VLM与开集检测评测Benchmark方案设计

> **来源文档**：方案设计文档/自建VLM&开集检测评测benchmark.pdf
> **评审维度**：系统和方案设计
> **更新时间**：2026-04

## 一、设计背景与目标

为后续本地部署VLM的能力测试提供一个**可直接运行、可量化评测、且覆盖多种典型视觉任务**的小型基准集。

核心用途：
1. 评估VLM在**家居场景**中的基础视觉理解能力
2. 为**开集目标检测**（open-vocabulary / open-set detection）相关实验提供统一的数据与评测接口支持

## 二、Benchmark设计思路

本benchmark并未设计为单一detection数据集，而是构建为**多任务统一评测基准**，同时覆盖以下几类任务能力：

| 任务类型 | 英文名 | 测试能力 |
|---------|-------|---------|
| 开集目标检测 | open-vocab detection | 模型是否能依据文本类别描述识别图像中的目标，并返回边界框 |
| 指代目标定位 | referring grounding | 模型是否能理解如"the left sofa"或"the right sofa"等指代表达，并准确定位 |
| 几何关系判断 | geometry relation | 模型是否能理解图像中多个目标之间的相对空间关系 |
| 房间类型分类 | room classification | 模型对整体场景语义的感知与判断能力 |
| 地图理解 | map understanding | 模型对ASM地图的理解能力 |

## 三、数据来源与组成

所有benchmark数据均来源于**homeobjects-3k**数据集，按不同任务从总数据集中采集不同数量图像进行后续处理。

### 3.1 各子集详细说明

| 数据子集 | 标注方式 | 样本构成 | 数据规模 |
|---------|---------|---------|---------|
| 带COCO标注的数据子集 | 标准COCO格式`_annotations.coco.json`，当前主要标注sofa类别（其他类别通过标注工具及其他数据集补充中） | 正样本、负样本、空样本、多目标物样本 | — |
| 带指代位置检测的数据子集 | 自建工具（手动标注bbox + VLM生成几何关系描述） | 正样本、负样本 | — |
| 用于描述几何关系的数据子集 | 从COCO数据子集中选择正样本及多样本数据，通过prompt组装形式锁定物品 | 正样本、多目标样本 | — |
| 房间描述/分类 | homeobjects-3k随机选取 | 多场景 | 50张图像 |
| 地图理解 | 仿真环境中ASM地图构建 | ASM地图图像 | — |

### 3.2 标注工具链

- **COCO标注**：标准标注工具生成`_annotations.coco.json`
- **指代表达标注**：自建工具，手动标注bbox后由VLM生成几何关系描述
- **几何关系数据**：复用COCO子集正样本，通过prompt组装形式生成

## 四、评测脚本与运行方式

```bash
# 运行标准VLM评测（无thinking模式）
python3 run_vlm_benchmark.py \
    --benchmark-dir homeobjects_mini_v1 \
    --base-url http://localhost:9002/v1 \
    --model Qwen3-VL-8B \
    --save-visualizations

# 运行Qwen3.5-4B评测（关闭thinking）
python3 run_vlm_benchmark_qwen35_4b.py \
    --benchmark-dir homeobjects_mini_v1 \
    --save-visualizations
```

## 六、设计挑战与创新点

1. **多任务统一评测**：突破单一detection数据集局限，将开集检测、指代定位、几何关系判断、房间分类、地图理解五类任务统一到一个benchmark中，全面评估VLM能力
2. **面向家居场景的数据构建**：基于homeobjects-3k构建面向家居导航的专项评测集，区别于通用benchmark，更贴合Kinbot实际部署场景
3. **自建标注工具链**：组合手动标注bbox + VLM生成几何关系描述的半自动标注方式，高效生成指代表达数据
