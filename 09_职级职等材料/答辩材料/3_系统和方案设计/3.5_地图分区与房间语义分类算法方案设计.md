# 地图分区与房间语义分类算法方案设计

> **来源文档**：技术规划文档/地图分区及房间语义地图演进.pdf
> **评审维度**：系统和方案设计
> **更新时间**：2026-04

## 一、设计背景

地图分区是Kinbot家居导航的基础能力，负责将障碍地图自动划分为房间区域并赋予语义标签。整体算法分为两个阶段：几何分区（将障碍地图分割为区域）和房间语义分类（为每个区域赋予语义标签）。

核心挑战：走廊等狭长区域与房间相邻时，当前算法无法有效区分，导致走廊被错误地合并到相邻房间中。

## 二、几何分区算法方案

### 2.1 基础分区流程

```
输入当前障碍图 ObstacleMap.visualize()
    ↓
prepare_partition_map（旋转 + 裁剪黑边 + 垂直翻转）
    ↓
_preprocess_map
  1. 二值化
  2. 提取墙体 wall_map
  3. 墙体闭运算增强
  4. 自由空间 processed_map
  5. 方波降噪
  6. 计算 distance_map
    ↓
_initial_segmentation（距离变换 + 高斯平滑 + 局部极大值种子 + watershed）
  └── 种子不足时退化到连通域分析
    ↓
初始区域 rooms
    ↓
_expand_regions（按自由空间约束迭代膨胀到边界，与其他区域重叠时则排除重叠部分）
    ↓
_remove_tiny_regions（删除小于最小房间面积的区域）
    ↓
_filter_abnormal_regions（过滤面积过大 / 边界异常 / 可疑大边界区域）
    ↓
_merge_regions（启发式合并相邻区域，依据面积、长宽比、公共边界长度等规则）
    ↓
_generate_result → 生成最终 result_map / rooms / statistics
    ↓
输出几何分区结果 partition_map / stats / vis
```

### 2.2 窄连接切分优化（针对走廊问题）

在`_merge_regions`之后新增`_split_regions_by_narrow_connections`步骤：

```
单个 region 距离变换
    ↓
提取厚核 core = dist <= _split_core_clearance，跳掉太小核心
    ↓
core 连通域分析
    ↓
存在多个大核心且核心中心距离足够远？
├── 是 → 在原region内以这些核心为seed做竞争式生长重分配 → 得到切分后的子区域
└── 否 → 保持原区域不变
    ↓
再次 _remove_tiny_regions 删除切分后的小碎块
    ↓
_generate_result → 输出最终几何分区结果
```

**算法核心思路**：通过距离变换找到区域内的"厚核"（远离边界的中心区域），如果一个区域存在多个相距较远的厚核，说明该区域可能包含了通过窄连接相连的不同空间（如走廊连接的两个房间），此时以各厚核为种子进行竞争式生长，将区域切分。

### 2.3 参数配置

所有阈值先用米/平方米定义，再按`pixels_per_meter`转换为像素，保证参数在不同分辨率地图上的一致性（`MapConfig`物理参数配置）。

## 三、房间语义分类算法方案

### 3.1 分类流程

```
输入：几何分区 rooms + partition_map + semantic objects
    ↓
_assign_objects_to_regions（将对象映射到对应region）
    ↓
逐个原始 region 计算 object_prior（根据对象类别统计房间先验分数）
    ↓
是否需要分类器？
├── 是 → 原始region分类：最大内切圆圆心 → 采集前右后左4张图 → classify_room
└── 否 → 直接使用 object_prior
    ↓
_fuse_room_predictions（融合 object_prior 与 classifier）
    ↓
_merge_adjacent_semantic_regions（合并相邻且同语义的原始region）
    ↓
_refine_merged_regions（对 merged mask 再取最大内切圆，再次采4张图进行 merged-region 分类）
    ↓
回写 merged 语义到原始 regions 并更新对象 area 字段
    ↓
生成可视化 + 保存结果 room_semantics JSON / PNG
```

### 3.2 关键设计细节

- **Object Prior**：根据区域内检测到的物体类别，利用先验知识统计房间类型概率（如检测到沙发→客厅概率高）
- **VLM分类器**：取区域最大内切圆圆心作为观测点，采集前/右/后/左4个方向图像送入VLM进行房间类型分类
- **融合策略**：将object_prior与VLM分类结果加权融合，提升分类鲁棒性
- **语义一致性合并**：相邻且同语义的区域合并后，对合并区域再次分类确认，避免合并引入错误

## 四、多场景效果验证

在4套真实户型上验证了分区+语义标注效果：

| 户型 | 分区效果 | 语义标注效果 |
|------|---------|-----------|
| 户型1 | 厨房、客厅、阳台、卫生间、卧室、小储藏间正确分区 | bedroom, kitchen, hallway, living_room, balcony |
| 户型2 | 卫生间、厨房、卧室、客厅正确分区 | bedroom, living_room, hallway |
| 户型3 | 卧室、阳台、客厅、厨房、卫生间、书房、入户正确分区 | bedroom, hallway, laundry, kitchen, living_room |
| 户型4 | 卧室、客厅、入户、厨房、卫生间正确分区 | bedroom, living_room, unknown, bathroom, hallway |

**已知问题**：窄连接切分优化对不同场景效果不稳定，需持续迭代。

## 五、设计挑战与创新点

1. **距离变换+厚核竞争式生长**：创新性地利用距离变换提取厚核并进行竞争式生长切分，解决走廊与房间无法区分的难题
2. **Object Prior + VLM分类双通道融合**：结合物体先验知识与视觉语言模型的场景理解能力，双通道融合提升房间分类准确率
3. **物理参数到像素参数的自动转换**：通过MapConfig统一管理，所有阈值以物理单位定义后自动转换，保证多分辨率地图的适配性
