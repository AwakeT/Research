# DToF传感器辅助脱困方案设计

> **来源文档**：方案设计文档/Dtof测试及tof脱困方案设计.pdf
> **评审维度**：系统和方案设计
> **更新时间**：2026-04

## 一、设计背景

当前脱困系统仅依赖震荡检测+固定动作序列，缺乏对前方障碍物的主动感知能力。DToF传感器可检测低矮障碍物（最矮4.3cm），但存在可信/不可信场景的边界条件。本方案将DToF数据融入脱困流程，利用tof辅助判断可通行方向，提升腿类障碍及机器横跨门槛场景下的脱困成功率。

## 二、DToF可信场景映射

基于实测数据，建立DToF数据可信与不可信的场景分类：

### 2.1 可信场景

| 场景 | 条件 | DToF表现 |
|------|------|---------|
| 近距离障碍物 | 撑手3°，距墙体4cm | 返回5cm，confidence=100，正常 |
| 远距离障碍物 | 撑手3°，距墙体20cm | 返回21-22cm，confidence=100，误差1-2cm |
| 空塑料瓶 | 撑手3°，移动水瓶 | 数据正常返回 |
| 含水水瓶 | 撑手3°，移动水瓶 | 数据正常返回 |
| 机器人运动中 | 撑手3°，前后左右移动 | 数据返回均正常 |

### 2.2 不可信场景

| 场景 | 条件 | DToF表现 | 原因 |
|------|------|---------|------|
| 小障碍物被遮挡 | 前方小障碍物后方有大障碍物 | 误差2-3cm，小障碍物不可识别 | tof光束被后方大障碍物干扰 |
| 奖杯/透明物体 | 奖杯旋转约25° | tof光束穿透奖杯，测量为后方障碍物距离 | 透明/反光材质穿透 |
| 侧方障碍物 | 右手距箱体0cm，左手15cm | 返回21cm（仅检测正前方） | **若箱体低于雷达，dtof与雷达均无法发现右轮处障碍物** |
| 光滑铁皮表面 | 与墙面角度25° | 折射现象，数值随前方物品改变 | 光滑表面折射 |
| 镜子 | 距镜子>0.47m（75°倾斜）或>1.4m（垂直） | data=1.5，confidence=0 | 镜面反射失效 |
| 玻璃 | 与玻璃垂线角度25° | data=0.09xx，confidence=50 | 玻璃透射失效，角度<25°时正常 |

## 三、方案一：震荡+DToF脱困方案

针对**腿类障碍及机器横跨门槛**场景的辅助脱困：

### 3.1 流程设计

```
1. 触发震荡事件 → 执行震荡脱困动作一：后退20cm
       ↓
2. 判断 dtof > 60cm？
   ├── 是 → 执行动作二：前进20cm → 进入原震荡脱困流程
   └── 否 → 执行辅助动作
       ↓
3. 辅助动作：先向左旋转30°后向右旋转60°
       ↓
4. 旋转过程中若 tof > 100cm，相互独立计数 max_count++
       ↓
5. max_count > 8？
   ├── 是 → 停止旋转 → 执行动作二：前进20cm → 进入原震荡脱困流程
   └── 否 → 保持当前最后机器人状态
```

### 3.2 辅助脱困动作详细描述

1. 以30°/s角速度向左旋转30度，可获得tof数据10个，若tof数据>80cm数量超过70%则判断该方向可通过
2. 若左转未能脱困，以30°/s角速度向右旋转60度，只计算后十个数据，若tof数据>80cm数量超过70%则判断该方向可通过
3. 若左右旋转均无法发现可通行方向，继续执行原震荡脱困动作二

## 四、方案二：震荡+DToF+OBS脱困方案

融合雷达扇形区域检测与DToF数据的增强方案：

### 4.1 流程设计

```
1. 震荡触发 → 执行震荡脱困动作一（后退20cm）
       ↓
2. 雷达扇形区域内无障碍物(45cm) && tof > 45cm？
   ├── 是 → 恢复导航
   └── 否 → 向左旋转45°
       ↓
3. 雷达扇形区域内无障碍物(45cm) && tof > 45cm？
   ├── 是 → 执行震荡脱困动作二，恢复导航
   └── 否 → 向右旋转90°
       ↓
4. 雷达扇形区域内无障碍物(45cm) && tof > 45cm？
   ├── 是 → 执行震荡脱困动作二，恢复导航
   └── 否 → 向左旋转45°，恢复执行旋转辅助脱困动作前状态
          → 执行脱困动作二，继续原震荡脱困动作
```

### 4.2 旋转辅助脱困动作详细步骤

1. **旋转辅助脱困动作一**：以30°/s向左旋转45°，判断过程中tof>45cm数量是否>80%且旋转完毕时雷达扇形范围内不存在障碍物
2. **旋转辅助脱困动作二**：以30°/s向右旋转90°，判断后45°过程中tof>45cm数量是否>80%且雷达扇形范围内无障碍物
3. **旋转辅助脱困动作三**：以30°/s向左旋转45°，恢复原状态，继续执行震荡脱困动作

## 五、场景模拟效果

1. **卡在桌/凳脚且震荡脱困动作一受阻**：辅助旋转动作可检测到可通行方向并完成脱困
2. **卡在凳脚内部且脱困动作一受阻**：四个凳脚包围的情况下，辅助旋转动作需在有限空间内寻找出口

## 六、待持续测试项

1. 平地发震荡时如何与家具腿类、门槛类进行区分，避免在平地进行旋转
2. 震荡脱困动作一完成后机器面前是否存在障碍物距离
3. 辅助脱困动作旋转角度，针对家具腿类、门槛类造成震荡时进行测试
4. 旋转过程如何判断该方向可通过，针对家具腿类、门槛类造成震荡时进行tof数据采集与分析
5. 如何避免因辅助脱困动作导致机器陷入更复杂困境（例如钻入凳子腿内部）

## 七、设计挑战与创新点

1. **传感器融合脱困**：首次将DToF传感器数据融入脱困流程，弥补雷达无法检测低矮障碍物的盲区
2. **基于旋转扫描的方向判断**：通过30°/s角速度旋转采集tof数据，利用统计阈值（70%/80%）判断可通行方向
3. **渐进式方案演进**：从纯tof辅助（方案一）到tof+雷达扇形区域融合（方案二），逐步提升判断可靠性
