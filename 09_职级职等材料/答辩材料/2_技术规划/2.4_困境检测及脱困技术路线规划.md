> **更新时间**：2026-04

## 一、技术模块现状

当前困境检测与脱困采用传感器数据操作为基础，通过各个传感器checker-事件-事件决策判断机器人状态。当前仅判断打滑与震荡事件，其中打滑事件仅做上报不做处理，只有震荡事件涉及具体脱困。

### 当前脱困流程

1. 脱困器初始化 → 动作1后退 → 检测是否成功
2. 等待动作完成 → 检测前方障碍 → 动作11左转45度 / 动作12右转90度 / 动作13左转45度
3. 动作2前进 → 动作3抬手前进 → 动作4-1/4-2撑手前进 → 动作5撑手后退 → 动作6撑手快前进
4. 动作无响应 / 未脱困 → 脱困失败与记录 → 最终动作撑手后退 → 重复脱困失败封锁

## 二、困境场景分类

通过家居场景实测，梳理了以下经典困境场景：

### 2.1 门槛类（最高可通过门槛高度为2cm）
- **推拉门门槛**：两侧高度一致（可通过）/ 两侧高度不一致（内部地板vs阳台瓷砖，存在高度差）
- **卫生间下沉门槛**：高度较高，通过困难
- **普通门槛**

### 2.2 家具类
- **桌腿、凳腿**：三种类桌腿或凳腿，机器在此处出现困境时具有普遍脱困能力
- **办公室转椅**：普遍需要1-2次完整脱困流程并结合探索给出的新目标点可完成脱困
- **罗马椅/小露营凳**：凳腿柱体宽度为3.5cm，机器前肢被抬起，无法正常执行脱困动作（最大可通过距离<2cm），需人为介入

### 2.3 房门
- 机器人通过旋转及传感器判断可完成脱困

### 2.4 玻璃门
- 透明材质传感器难以检测

### 2.5 地毯
- 机器在地毯上可能产生打滑

### 2.6 柜子
- 柜子底部间隙可能导致机器卡住

## 三、技术发展方向

### 3.1 从规则到模型化

当前纯规则驱动的脱困流程存在局限性：
- 动作序列固定，无法根据具体场景自适应
- 缺乏对障碍物类型的识别能力
- 无法利用视觉信息辅助判断

**规划方向**：引入视觉模型辅助困境检测，结合VLM的场景理解能力判断困境类型并选择最优脱困策略。

### 3.2 传感器融合

- DToF + 雷达 + 视觉的多源融合
- 利用DToF检测低矮障碍物（详见Dtof测试方案）
- 利用视觉检测透明/反光障碍物

## 四、技术路径与目标

| 阶段 | 目标 | 关键技术 |
|------|------|---------|
| 当前 | 基于传感器的规则脱困 | 震荡检测+固定动作序列 |
| 短期 | DToF辅助困境检测 | dtof可信场景映射+脱困方案融合 |
| 中期 | 视觉辅助困境识别 | VLM场景分类+自适应脱困策略 |
| 远期 | 模型化脱困 | 学习型脱困策略，根据场景自动选择最优动作 |

## 五、规划执行成果

- 完成了家居场景下经典困境的系统性分类（6大类）
- 通过实测明确了各场景的脱困能力边界（最大可通过高度2cm、最大可通过距离<2cm）
- 完成了DToF传感器的可信场景测试（详见Dtof测试与方案设计文档）
- 形成了从规则到模型化的渐进式技术路线
