# VLM基座模型选型规划：Teacher-Student蒸馏路线

> **来源文档**：调研文档/小模型效果对比V2.pdf + 调研文档/基座模型选型分析.pdf
> **评审维度**：技术规划
> **更新时间**：2026-04

## 一、技术模块现状

Kinbot VLN→NFM方案的核心架构为：27B Qwen VL Teacher → 多轮蒸馏 4B Student。当前面临的关键决策是Teacher和Student的基座模型选择。

**现有约束**：
- Student需部署到S100 Pro / RK3588（量产）或Jetson Orin AGX（研究）
- Token预算严格控制在450-900 token/次（按任务级别分档）
- 输入契约：1组双目 + 2个单目（PDCP量产基线）

## 二、技术趋势判断

1. **Qwen3-VL系列已停止演进**，官方不再发布新版本；Qwen3.5系列是长期主线方向
2. **Early Fusion架构**（Qwen3.5）正在取代Encoder-Decoder架构（Qwen3-VL），视觉和文本在统一Transformer中原生交互
3. **混合注意力架构**（Gated DeltaNet + Softmax Attention）大幅降低KV cache压力，更适合端侧推理
4. **Thinking模式**在通用benchmark上表现优异，但在实际检测任务中No-Thinking更优

## 三、选型结论与规划

### 3.1 核心选型

| 角色 | 推荐模型 | 理由 |
|------|---------|------|
| Teacher (27B级) | **Qwen3.5-27B** | 原生多模态、同架构族蒸馏路径最直接 |
| Student (4B级) | **Qwen3.5-4B (no-thinking)** | homeobjects实测全面领先，空间关系理解大幅优于Qwen3-VL-8B |

### 3.2 蒸馏路径规划

**蒸馏目标**：将Teacher"想清楚"的结论教给Student"做快速"

- Teacher用thinking来"想清楚"（仅限离线单次推理）
- Student用no-thinking来"做快速"
- 推理链是Teacher的内部工具，不是Student的输出目标

**蒸馏损失设计**：
```
0.4×region_ranking + 0.3×location_cls + 0.2×next_action + 0.1×confidence
```

### 3.3 训练范式

- Teacher多阶段训练 + 蒸馏到Student
- 能力分层：P0空间理解/地图/基础空间记忆 → P1多视角/几何/RL → P2个性化记忆
- 频率架构：低频主策略层 + 高频轻量层

## 四、短期技术路径

1. **第一阶段**：基于Qwen3.5-27B搭建Teacher训练pipeline，验证grounding微调效果
2. **第二阶段**：构建蒸馏数据集（Teacher推理结果作为Student的训练标签）
3. **第三阶段**：Qwen3.5-4B Student蒸馏训练，验证端侧部署效果
4. **长期储备**：跟踪Qwen3.5系列后续版本，确保架构兼容性

- 完成了Qwen3-VL vs Qwen3.5的系统性架构对比和benchmark验证
- 确定了Teacher-Student蒸馏路线的基座模型选型
- 形成了P0-P1-P2能力分层的渐进式训练规划
