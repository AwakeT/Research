> **来源文档**：技术规划文档/地图分区及房间语义地图演进.pdf
> **评审维度**：技术规划
> **更新时间**：2026-03

## 一、技术模块现状

地图分区是Kinbot家居导航的基础能力，负责将障碍地图自动划分为房间区域并赋予语义标签。当前已实现基础分区算法（watershed+膨胀+合并），但存在走廊等狭长区域与房间相邻时无法区分的问题。

### 当前算法流程

1. 输入障碍图 → `prepare_partition_map`（旋转+裁剪黑边+垂直翻转）
2. `_preprocess_map`（二值化/提取墙体/墙体闭运算/生成自由空间）
3. `_initial_segmentation`（距离变换+局部极大值种子点）→ 种子点足够则watershed，否则连通域分析
4. `_expand_regions`（区域膨胀扩展到边界）
5. `_remove_tiny_regions` + `_filter_abnormal_regions`
6. `_merge_regions`（按面积/狭长程度/公共边界长度启发式合并）
7. `_generate_result` → 输出分区结果

### 房间语义分类流程

1. 输入几何分区rooms + partition_map + semantic objects
2. `_assign_objects_to_regions`（将对象映射到对应region）
3. 逐个region计算object_prior（根据对象类别统计房间先验分数）
4. 需要分类器时：取最大内切圆圆心 → 采集前右后左4张图 → classify_room
5. `_fuse_room_predictions`（融合object_prior与classifier）
6. `_merge_adjacent_semantic_regions`（合并相邻且同语义的region）
7. `_refine_merged_regions`（对merged mask再次分类）
8. 回写语义到原始regions → 输出room_semantics JSON/PNG

## 二、问题诊断与演进方向

### 核心问题
走廊等狭长区域与房间相邻时，当前算法无法有效区分，导致走廊被错误地合并到相邻房间中。

### 演进路线

**阶段一：基于当前框架优化**
- 在`_merge_regions`后新增`_split_regions_by_narrow_connections`步骤
- 对大区域做窄连接切分后处理：
  1. 单个region距离变换
  2. 提取厚核（core = dist <= `_split_core_clearance`），跳掉太小核心
  3. core连通域分析：存在多个大核心且核心中心距离足够远 → 以这些核心为seed做竞争式生长重分配
  4. 再次`_remove_tiny_regions`删除切分后的小碎块
  5. `_generate_result` → 输出最终结果

**阶段二**：引入更多几何特征（长宽比、开口方向等）辅助走廊识别

**阶段三**：结合VLM的语义理解能力做端到端房间分类

## 三、技术路径与目标

| 阶段 | 目标 | 关键技术 |
|------|------|---------|
| 当前 | 基础分区+语义标注可用 | watershed + object_prior + VLM分类 |
| 优化 | 走廊/过道区分准确 | 窄连接切分 + 竞争式生长 |
| 远期 | 多场景稳定 | 几何+语义融合、自适应参数 |

## 四、规划执行成果

- 完成了基础分区+语义标注全流程实现
- 在4套真实户型上验证了效果（详见算法验证文档）
- 设计并实现了窄连接切分算法，针对走廊问题进行了优化（效果仍存在对不同场景不稳定的情况，需持续迭代）
