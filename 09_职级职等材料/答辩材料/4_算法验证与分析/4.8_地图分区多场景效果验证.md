# 地图分区多场景效果验证

> **来源文档**：技术规划文档/地图分区及房间语义地图演进.pdf
> **评审维度**：算法验证与分析
> **更新时间**：2026-04

## 一、验证目标

在多套真实户型上验证地图分区算法（几何分区+房间语义分类）的效果，评估算法在不同户型结构下的泛化能力和准确性。

## 二、实验配置

### 2.1 算法流程

```
障碍地图 → 预处理（二值化+墙体提取+方波降噪）
    → 初始分割（距离变换+watershed）
    → 区域膨胀 → 去除小区域 → 过滤异常区域
    → 启发式合并 → 窄连接切分
    → 语义分类（Object Prior + VLM分类融合）
    → 输出分区+语义标注结果
```

### 2.2 测试数据

- **数据来源**：4套真实家居户型的障碍地图
- **户型类型**：覆盖不同面积、不同房间数量、不同布局结构

## 四、问题分析

### 4.1 几何分区层面

**整体表现良好**：4套户型的几何分区均能正确识别主要房间边界。

**已知问题**：窄连接切分优化对不同场景效果不稳定，需持续迭代。具体表现为：
- 走廊与房间的边界在部分户型中不够清晰
- 窄连接切分的`_split_core_clearance`参数需要针对不同户型进行调优

### 4.2 语义分类层面

| 问题类型 | 示例 | 分析 |
|---------|------|------|
| 误分类 | laundry（实际可能为卫生间） | Object Prior中洗衣机类物品导致误判 |
| 未识别 | unknown标注 | VLM对该区域4方向图像无法判断房间类型 |
| 走廊识别 | hallway频繁出现 | 走廊区域的语义判断较为准确 |

### 4.3 Object Prior vs VLM分类的融合效果

- 当区域内检测到明确特征物品（床→卧室、沙发→客厅）时，Object Prior主导，分类准确
- 当区域内特征物品不明确时，VLM分类补充，但存在不确定性
- 融合策略整体提升了分类鲁棒性，但在物品误检时可能被误导

## 五、参数敏感性观察

| 参数 | 影响 | 建议 |
|------|------|------|
| 最小房间面积阈值 | 过大会吞掉小房间（储藏间），过小会保留噪声区域 | 根据户型面积自适应调整 |
| 窄连接切分厚核阈值 | 影响走廊与房间的切分边界 | 需针对不同走廊宽度适配 |
| 合并规则中的面积/长宽比门限 | 影响小区域是否被合并到相邻大区域 | 当前启发式规则基本可用 |

## 六、总结

1. **几何分区算法在真实户型上泛化性良好**：4套户型均能正确分区，主要房间边界识别准确
2. **语义分类整体可用但存在边界case**：unknown标注和个别误分类需通过增加VLM分类置信度阈值和补充Object Prior类别映射来改善
3. **窄连接切分是当前最大改进空间**：效果不稳定，需在更多户型上测试并调优参数
4. **Object Prior + VLM融合策略的价值得到验证**：双通道互补显著优于单通道，但需防范物品误检对Prior的干扰
