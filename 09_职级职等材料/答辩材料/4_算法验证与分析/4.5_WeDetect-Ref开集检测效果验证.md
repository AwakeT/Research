# WeDetect-Ref开集检测效果验证

> **来源文档**：需求评估与调研文档/大模型+开集检测调研.pdf
> **评审维度**：算法验证与分析
> **更新时间**：2026-04

## 一、实验背景

WeDetect-Ref是WeDetect的扩展版本，增加了指代表达理解（Referring Expression Comprehension）能力。本测试验证其在家居场景下的开集检测效果及资源开销。

## 二、实验配置

### 2.1 模型信息

| 参数 | 配置 |
|------|------|
| 模型 | WeDetect-Ref |
| 架构 | WeDetect + 指代表达理解模块 |
| 能力 | 开集目标检测 + 指代表达定位 |

### 2.2 硬件环境

| 参数 | 数值 |
|------|------|
| GPU | NVIDIA RTX 4090 |
| 显存占用 | 7180 MiB |
| 推理速度 | 0.434s/张 |

## 三、检测效果验证

### 3.1 正确检测案例

在家居场景标准物品（沙发、桌子、椅子、床等）上，WeDetect-Ref能够：
- 正确定位目标物品并返回bbox
- 支持指代表达如"the left sofa""the red chair"等
- 置信度合理，bbox精度可接受

### 3.2 错误检测案例

| 错误类型 | 示例 | 分析 |
|---------|------|------|
| 漏检 | 小物品（遥控器、杯子）未检测到 | 小物品特征不够显著 |
| 误检 | 背景纹理被误识别为物品 | 开集检测对未知类别的泛化能力有限 |
| 定位偏差 | bbox明显偏离目标中心 | 指代表达歧义或特征提取不准确 |

## 四、资源开销分析

### 4.1 与其他方案对比

| 方案 | 显存占用 | 推理速度 | 备注 |
|------|---------|---------|------|
| WeDetect-Ref | 7180 MiB | 0.434s | 支持指代表达 |
| Grounding-DINO + MobileSAM | 较低 | 较快 | 需要两阶段 |
| LLMDet | 极高（需要大模型） | 较慢 | 端到端但资源密集 |

### 4.2 端侧部署可行性

- **显存7180MiB**：在RTX 4090上可运行，但端侧设备（如Jetson系列）难以承载
- **推理速度0.434s**：对于实时导航场景偏慢，但作为非实时的物品搜索场景可接受

## 五、结论

1. **检测能力基本满足需求**：在家居常见物品上表现合理，支持指代表达是亮点
2. **资源开销偏高**：7180MiB显存和0.434s延迟限制了端侧实时部署
3. **与当前ASM方案的对比**：Grounding-DINO+MobileSAM的两阶段方案在资源效率上更优，WeDetect-Ref的优势在于原生支持指代表达
4. **建议定位**：作为备选方案关注，当指代表达成为核心需求时再考虑引入
