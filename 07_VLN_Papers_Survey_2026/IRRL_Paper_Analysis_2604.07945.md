# IRRL：面向真实世界学习的增量残差强化学习社交导航

> **论文标题**: Incremental Residual Reinforcement Learning Toward Real-World Learning for Social Navigation
> **arXiv**: 2604.07945v1
> **发表时间**: 2026年4月
> **作者**: Haruto Nagahisa, Kohei Matsumoto, Yuki Tomita, Yuki Hyodo, Ryo Kurazume（九州大学）
> **关键词**: Social Navigation, Incremental Learning, Residual RL, Real-World RL, Edge Device

---

## 一、论文概述

### 1.1 研究背景与动机

随着移动机器人在公共空间（商场、医院、仓库）的部署日益增多，**社交导航（Social Navigation）** 成为关键课题——机器人不仅需要到达目标，还要在行人密集环境中遵守社交规范（如保持安全距离、不打断行人路径、不强行穿插）。

传统方法的局限性：
- **基于规则的方法**（如 DWA、TEB）：无法处理复杂的人-机交互模式
- **标准深度强化学习（Standard DRL）**：依赖大型经验回放缓冲区（Replay Buffer）和大批量更新，计算资源需求高，难以在边缘设备上部署
- **仿真到真实（Sim-to-Real）迁移**：行人行为和社交规范在不同文化、不同场景中差异巨大，仿真无法覆盖所有真实场景

**核心问题**：如何让机器人在真实物理环境中、在边缘设备上、以低计算成本**实时学习和适应**行人行为？

### 1.2 核心贡献

本文提出 **IRRL（Incremental Residual RL）** 框架，整合三项关键技术：

1. **增量学习（Incremental Learning）**：无需经验回放缓冲区，仅使用最新的单步转移数据进行更新
2. **残差强化学习（Residual RL）**：冻结的基础策略（SFM）+ 可学习的残差修正
3. **稳定化技术**：倒数第二层归一化（Penultimate Normalization）+ TD误差缩放（TD Error Scaling）+ 自适应温度调节

**论文声称这是首次将增量学习方法应用于社交导航领域。**

## 二、方法详解

### 2.1 整体架构

IRRL 的核心思想可以用一个公式表达：

```
最终动作 = 基础策略动作(frozen) + 残差策略动作(learned)
  a_t    =    a_t^base         +     a_t^res
```

整体流程：
1. **基础策略（Base Policy）**：Social Force Model（SFM），冻结不更新，提供安全保底行为
2. **残差策略（Residual Policy）**：Actor-Critic 网络，使用 GATv2 聚合行人特征，输出残差修正量
3. **增量更新**：每一步只用当前 transition `(s_t, a_t, r_t, s_{t+1})` 更新网络，无需存储历史数据

### 2.2 基础策略：社会力模型（SFM）

#### 术语解释：Social Force Model（SFM，社会力模型）

SFM 是 1995 年由 Helbing 和 Molnár 提出的经典行人动力学模型。其核心思想是将行人运动建模为**多种"力"的合力**：

| 力的类型 | 含义 | 类比 |
|---------|------|------|
| **目标吸引力（Attractive Force）** | 指向目标点的驱动力 | 类似重力，将行人"拉向"目标 |
| **行人排斥力（Repulsive Force from Pedestrians）** | 与其他行人之间的排斥力 | 类似同极磁铁的排斥 |
| **障碍物排斥力（Repulsive Force from Obstacles）** | 与墙壁/障碍物的排斥力 | 防止碰撞的"势场" |

SFM 本质上是一种**人工势场法（Artificial Potential Field）**的变体，优点是计算简单、物理直觉清晰，缺点是：
- 容易陷入局部极小值（如狭窄走廊对面来人时）
- 无法学习复杂的社交规范（如在走廊靠右走）
- 参数需要手动调节

在 IRRL 中，SFM 作为**冻结的基础策略**，提供合理但非最优的保底行为，残差策略在此基础上进行修正。

### 2.3 残差策略网络架构

#### 2.3.1 Actor-Critic 架构

#### 术语解释：Actor-Critic（演员-评论家）

Actor-Critic 是强化学习中的一类经典架构，包含两个网络：

| 组件 | 角色 | 输入 | 输出 |
|------|------|------|------|
| **Actor（演员）** | 策略网络，决定"做什么" | 聚合后的行人特征 + 基础策略动作 | 高斯分布的均值和方差（动作分布） |
| **Critic（评论家）** | 价值网络，评估"做得好不好" | 聚合后的行人特征 + 基础动作 + 残差动作 | Q值（状态-动作价值） |

Actor 输出高斯策略 `π_θ(a|s)`，即给定状态 s 下动作 a 的概率分布。Critic 输出 Q 值 `Q_δ(s, u)`，评估在状态 s 下执行组合动作 u（基础+残差）的期望累积奖励。

#### 2.3.2 GATv2 特征聚合

#### 术语解释：GATv2（Graph Attention Network v2，图注意力网络第二版）

GATv2 是 Brody 等人（2022）提出的改进版图注意力网络。为什么需要图神经网络？

**问题**：行人数量是动态变化的——可能有0个、3个、10个行人。传统全连接网络需要固定大小的输入，无法处理可变数量的行人。

**解决方案**：将场景建模为图（Graph）：
- **节点（Node）**：每个行人是一个节点，特征包括相对位置和相对速度
- **边（Edge）**：行人之间的关系（注意力权重）

GATv2 相比原始 GAT 的改进：
- 原始 GAT：注意力系数的计算是**静态排序**的（static attention），即对于固定的节点特征排列，注意力分数始终相同
- GATv2：使用**动态注意力**，注意力系数能真正依赖于查询节点和键节点的组合，表达能力更强

在 IRRL 中，GATv2 将可变数量的行人特征聚合为**固定维度的特征向量**，输入给 Actor 和 Critic 网络。

#### 2.3.3 状态表示

每个行人 i 的状态表示为：

```
s_i = [p_i^x, p_i^y, v_i^x, v_i^y]
```

其中：
- `p_i^x, p_i^y`：行人相对于机器人的位置（在机器人坐标系下）
- `v_i^x, v_i^y`：行人相对于机器人的速度

机器人自身状态：
- 目标的相对坐标 `[g^x, g^y]`
- 到目标的距离和角度

所有行人特征通过 GATv2 聚合后，与机器人自身状态拼接，输入 Actor/Critic 网络。

### 2.4 增量学习机制

#### 术语解释：Incremental Learning（增量学习）

增量学习是指模型**逐步从新数据中学习，而不需要存储和重放历史数据**。与标准 DRL 的对比：

| 特性 | 标准 DRL（带 Replay Buffer） | 增量学习（IRRL） |
|------|---------------------------|----------------|
| 数据存储 | 存储数万~数百万条历史经验 | 不存储，只用当前单步数据 |
| 更新方式 | 从缓冲区随机采样小批量更新 | 每步直接更新 |
| 内存需求 | 高（缓冲区可能占数GB） | 极低 |
| 计算成本 | 高（批量采样+更新） | 低 |
| 数据效率 | 高（同一数据可被多次使用） | 低（每条数据只用一次） |
| 适合场景 | 离线/仿真训练 | 在线/真实世界训练 |

#### 术语解释：Replay Buffer（经验回放缓冲区）

Replay Buffer 是标准 DRL 的核心组件之一，本质上是一个**先进先出（FIFO）队列**，存储形如 `(s_t, a_t, r_t, s_{t+1})` 的经验元组。训练时从中随机采样小批量数据，目的是：
1. **打破时序相关性**：连续的经验高度相关，随机采样可以得到更独立的数据
2. **提高数据利用率**：同一条经验可以被多次采样训练

**IRRL 完全不使用 Replay Buffer**，代价是失去了上述两个优势，但换来了：
- 内存占用极低，适合边缘设备
- 无需管理缓冲区大小和采样策略
- 天然适合在线学习

#### 术语解释：AVG（Action Value Gradient）

AVG 是 Vasan 等人（2024）提出的一种**流式（streaming）深度强化学习算法**，专门为无 Replay Buffer 的场景设计。IRRL 以 AVG 为基础构建其增量学习过程。

AVG 的核心特点：
- 基于 Actor-Critic 架构
- 使用**熵最大化（Entropy Maximization）** 鼓励探索
- 专门设计了归一化和缩放技术来应对无 Replay Buffer 时的训练不稳定问题

#### 术语解释：Entropy Maximization（熵最大化）

在 RL 中，策略的熵衡量了**动作选择的随机性**：
- 高熵 = 动作分布均匀 = 探索性强
- 低熵 = 动作集中在少数选择 = 利用性强

熵最大化 RL 的目标函数不仅最大化累积奖励，还附加一个熵正则项：

```
目标 = 最大化 [Σ (r_t + α × H(π(·|s_t)))]
```

其中 α 是温度参数（见下文），H 是策略的熵。这防止策略过早收敛到确定性行为，保持适度探索。

### 2.5 稳定化技术

增量学习（无 Replay Buffer）天然面临训练不稳定的问题，IRRL 采用三项关键技术来缓解：

#### 2.5.1 倒数第二层归一化（Penultimate Normalization）

#### 术语解释：Penultimate Normalization（倒数第二层归一化）

这是 Bjorck 等人（2022）提出的技术。"Penultimate"意为"倒数第二"。

**做法**：对 Actor 和 Critic 网络的**倒数第二层（即输出层之前的最后一个隐藏层）** 的特征向量进行 L2 归一化：

```
ψ(S) = v_ψ(S) / ||v_ψ(S)||₂
```

其中 `v_ψ(S)` 是倒数第二层的特征向量。

**为什么有效**：
- 在增量学习中，由于没有 Replay Buffer 的数据多样性，网络特征可能发生剧烈漂移
- L2 归一化将特征向量约束在单位球面上，防止特征量级爆炸
- 减少了**灾难性遗忘（Catastrophic Forgetting）** 的风险——网络不会因为新数据的学习而完全覆盖旧知识

#### 2.5.2 TD 误差缩放（TD Error Scaling）

#### 术语解释：TD Error（时序差分误差）

TD Error 是强化学习中最基础的信号之一，衡量**预测价值与实际观测之间的差距**：

```
δ_t = r_t + γ × V(s_{t+1}) - V(s_t)
```

其中：
- `r_t`：当前步获得的即时奖励
- `γ`：折扣因子（0~1之间，衡量对未来奖励的重视程度）
- `V(s)`：状态价值函数（Critic 的估计）

**TD Error Scaling 做法**：将原始 TD 误差除以其**在线估计的标准差**：

```
δ_scaled = δ / σ_δ
```

其中 σ_δ 通过 Schaul 等人（2021）的方法在线估计（无需缓冲区）。

**为什么有效**：
- 无 Replay Buffer 时，TD 误差的方差可能非常大
- 缩放后的 TD 误差量级稳定，梯度更新更平滑
- 类似于 BatchNorm 对特征的归一化效果，但作用于 TD 误差

#### 2.5.3 自适应温度调节

温度参数 α 控制探索与利用的平衡（见熵最大化部分）。IRRL 使用 Haarnoja 等人（2018）的方法自动调节 α：
- 当策略熵低于目标熵时，增大 α → 鼓励更多探索
- 当策略熵高于目标熵时，减小 α → 鼓励更多利用

### 2.6 奖励函数设计

IRRL 使用的奖励函数：

```
r_t = {
  -1.0                           如果发生碰撞（d_c < 0）
  (d_c - 0.2) × 0.125           如果 d_c < 0.2（接近碰撞）
  其他情况为 0
}
```

其中：
- `d_c`：机器人与最近行人/障碍物之间的**最小间隔距离（minimum separation distance）**
- `p_t`：机器人当前位置
- `p^g`：目标位置

另外，论文使用了一个**势能型奖励**来引导机器人接近目标：

```
r_t += ||p_{t-1} - p^g|| - ||p_t - p^g||
```

即如果机器人在这一步离目标更近了，获得正奖励。

**设计特点**：
- 碰撞惩罚为 -1.0（最严重）
- 接近碰撞有渐变惩罚（0.2m 安全边际）
- 没有显式的"社交距离"奖励，社交行为主要依赖 SFM 基础策略提供

### 2.7 为什么选择残差 RL？

#### 术语解释：Residual RL（残差强化学习）

残差 RL 由 Silver 等人（2018）和 Johannink 等人（2019）提出，核心思想是：

**不从零学习完整策略，而是在一个已有的基础策略上学习"修正量"。**

```
π_final(s) = π_base(s) + π_residual(s)
```

| 对比维度 | 从零学习（Standard RL） | 残差学习（Residual RL） |
|---------|---------------------|---------------------|
| 初始行为 | 随机动作（危险） | 基础策略的合理行为（安全） |
| 学习难度 | 需要学习完整的导航策略 | 只需学习偏差/修正量 |
| 安全保证 | 训练初期无任何安全保证 | 基础策略提供保底安全 |
| 训练效率 | 需要大量数据 | 更高效（搜索空间更小） |
| 真实世界适用性 | 差（初期太危险） | 好（一开始就相对安全） |

在 IRRL 中：
- **基础策略 = SFM**：冻结不更新，提供避障保底
- **残差策略 = 学习的 Actor 网络**：输出对 SFM 动作的修正
- 即使残差策略输出为零，机器人仍按 SFM 行为运动，不会发生危险

### 2.8 简化设计决策

论文明确排除了以下标准 DRL 组件：
- **Target Networks（目标网络）**：标准 DRL 中用于稳定训练的延迟更新网络，IRRL 认为其对增量学习无益
- **Double Q-learning（双重 Q 学习）**：用于解决 Q 值过估计问题，IRRL 通过 TD 误差缩放替代

这些简化降低了计算开销和内存需求，使在边缘设备上运行成为可能。

## 三、实验设计与结果

### 3.1 仿真实验

#### 3.1.1 环境设置

使用 **CrowdNav** 仿真器，场景为**圆形交叉（circle-crossing）**：5个行人在圆形区域内交叉行走。

#### 术语解释：CrowdNav

CrowdNav 是社交导航领域广泛使用的仿真平台，特点：
- 2D 俯视图环境
- 行人按预设策略（SFM 或 ORCA）运动
- 机器人需要在不碰撞的情况下穿越行人群到达目标

#### 术语解释：ORCA（Optimal Reciprocal Collision Avoidance，最优互惠碰撞规避）

ORCA 是 van den Berg 等人（2011）提出的多智能体避碰算法：
- 每个智能体计算"速度障碍（Velocity Obstacle）"
- 选择在安全速度集合中最接近期望速度的速度
- 假设所有智能体都使用 ORCA → 互相协调避让

与 SFM 对比：
| 特性 | SFM | ORCA |
|------|-----|------|
| 原理 | 力模型（连续） | 速度空间约束（优化） |
| 协调性 | 单向（每个行人独立计算） | 互惠（假设对方也在避让） |
| 对称性 | 非对称 | 对称 |
| 行为表现 | 更像真实行人（有犹豫） | 更高效但机械化 |

论文测试了两种行人策略以评估 IRRL 的泛化能力。

#### 3.1.2 对比基线

**三类基线方法**：

**① 基础策略（不学习）**：
- SFM：纯社会力模型

**② 带 Replay Buffer 的 RL（标准方法）**：
- **SAC（Soft Actor-Critic）**：最流行的连续动作空间 RL 算法，使用 Replay Buffer + 双 Q 网络 + 自动温度调节
- **TD3（Twin Delayed DDPG）**：使用双 Critic + 延迟策略更新来稳定训练
- **PPO（Proximal Policy Optimization）**：信赖域方法，通常用于离散动作空间但也可用于连续空间

#### 术语解释：SAC（Soft Actor-Critic）

SAC 是 Haarnoja 等人（2018）提出的算法，特点：
- 最大化奖励 + 策略熵（off-policy 熵最大化 RL）
- 使用双 Q 网络取最小值防止过估计
- 自动调节温度参数
- 被广泛认为是连续动作空间最稳定的 RL 算法之一

#### 术语解释：TD3（Twin Delayed DDPG）

TD3 是 Fujimoto 等人（2018）对 DDPG 的改进：
- **Twin**：使用两个 Critic 网络，取最小值
- **Delayed**：Actor 更新频率低于 Critic
- **确定性策略**：Actor 输出确定的动作（不是分布）

#### 术语解释：PPO（Proximal Policy Optimization）

PPO 是 Schulman 等人（2017）提出的 on-policy 算法：
- 使用裁剪的目标函数限制策略更新幅度
- 训练稳定但数据效率较低（每批数据只用一次）
- 在 Atari、机器人等领域广泛使用

**③ 增量 RL（无 Replay Buffer）**：
- **Stream AC(λ)**：流式 Actor-Critic，使用资格迹（eligibility trace）
- **SAC-1**：SAC 的 batch size=1 变体（仅用最新一条数据更新）
- **TD3-1**：TD3 的 batch size=1 变体

所有 RL 方法（②和③）均在**残差 RL 框架内**使用 SFM 作为基础策略，确保公平对比。

#### 3.1.3 仿真结果

**TABLE I 关键数据**：

| 方法 | SFM行人成功率↑ | SFM行人碰撞率↓ | ORCA行人成功率↑ | ORCA行人碰撞率↓ |
|------|-------------|-------------|-------------|-------------|
| SFM (base) | 0.806 | 0.194 | 0.786 | 0.214 |
| SAC (w/ buffer) | 1.000 | 0.000 | 0.926 | 0.003 |
| TD3 (w/ buffer) | 1.000 | 0.000 | 0.993 | 0.003 |
| PPO (w/ buffer) | 0.966 | 0.034 | 0.983 | 0.003 |
| **IRRL (w/o buffer)** | **0.988** | **0.003** | **0.984** | **0.005** |
| Stream AC(λ) | 0.915 | 0.086 | 0.968 | 0.003 |
| SAC-1 | 0.825 | 0.192 | 0.914 | 0.084 |
| TD3-1 | 0.873 | 0.118 | 0.910 | 0.058 |

**核心发现**：
1. **IRRL 无需 Replay Buffer 就接近了有 Buffer 的方法（SAC/TD3）的性能**，成功率 0.988 vs 1.000
2. **IRRL 在所有无 Buffer 方法中表现最好**，且标准差最小（最稳定）
3. Stream AC(λ) 和 TD3-1 在某些种子下出现了**灾难性失败（catastrophic failure）**——训练过程中 return 骤降至零且无法恢复
4. **IRRL 从未出现灾难性失败**

### 3.2 消融实验：残差 RL 的贡献

论文对比了 IRRL（有残差 RL）vs 纯增量学习（无残差 RL，从零学习）：

**关键发现**：
- 有残差 RL：收敛更快、return 更高、跨种子方差更小
- 无残差 RL：部分种子完全无法提升 return，出现高方差
- 结论：**残差 RL 显著提升了增量学习的效率和稳定性**

### 3.3 真实世界实验

#### 3.3.1 硬件平台

| 组件 | 型号/规格 |
|------|---------|
| 底盘 | 麦克纳姆轮移动平台（Mecanum-wheeled rover） |
| LiDAR | 3D LiDAR（AR-XT-32-a） |
| 计算单元 | **Jetson AGX Orin** |
| 软件栈 | **ROS 2 Humble** |
| 行人检测 | DR-SPAAM |
| 定位 | MCL（蒙特卡洛定位）+ Expansion Resetting |

#### 术语解释：Jetson AGX Orin

NVIDIA Jetson AGX Orin 是一款面向机器人和边缘 AI 的高性能嵌入式计算平台：
- GPU：Ampere 架构，2048 个 CUDA 核心
- CPU：12核 ARM Cortex-A78AE
- 内存：32/64GB LPDDR5
- 功耗：15W~60W（可配置）
- 算力：最高 275 TOPS（INT8）

这是目前机器人领域最常用的高端边缘计算平台，IRRL 能在其上实时运行表明其计算需求确实较低。

#### 术语解释：DR-SPAAM（Deep Range-SPAAM）

DR-SPAAM 是 Jia 等人（2020）提出的基于深度学习的激光雷达行人检测算法：
- 输入：2D 激光扫描点云
- 输出：行人位置
- 特点：轻量级，可在嵌入式设备上实时运行

论文发现 DR-SPAAM 原始模型会将静态物体误检为行人，因此添加了**额外的过滤步骤**来排除静态检测结果。

#### 术语解释：MCL（Monte Carlo Localization，蒙特卡洛定位）

MCL 又称粒子滤波定位，是移动机器人最广泛使用的定位算法之一：
- 使用大量**粒子（particle）** 表示机器人可能的位置分布
- 每个粒子是一个位姿假设 `(x, y, θ)`
- 通过传感器观测（LiDAR 与地图匹配）更新粒子权重
- 重采样保留高权重粒子

**Expansion Resetting**：当机器人"绑架"（定位丢失）时，在更大范围内重新撒粒子以恢复定位。

#### 3.3.2 混合环境（Hybrid Environment）

持续与真实行人交互不切实际（人会累），因此论文设计了**混合环境**：

- **物理行人**：真实人类在场地中走动
- **虚拟行人**：在 RViz2 中以绿色圆柱体可视化，机器人将其视为真实行人进行避让

实验流程：
1. **预训练**：在仿真中训练 100,000 episodes（SFM 控制的合作行人）
2. **初始评估**：30 episodes（圆形交叉场景）
3. **在线学习**：100 episodes（混合环境，包含虚拟+物理行人）
4. **最终测试**：在物理行人环境中评估

#### 3.3.3 真实世界结果

**TABLE II 关键数据**：

| 环境 | 阶段 | 成功率↑ | 碰撞率↓ | 平均执行时间(s) | 平均 Return↑ |
|------|------|---------|---------|--------------|-------------|
| 虚拟行人 (50 trials) | Before | 0.520 | 0.440 | 6.56 | 0.256 |
| 虚拟行人 (50 trials) | After | 0.820 | 0.180 | 7.30 | 0.537 |
| 物理行人 (5 trials) | Before | 0.200 | 0.600 | 8.75 | 0.129 |
| 物理行人 (5 trials) | After | 0.600 | 0.400 | 9.15 | 0.299 |

**核心发现**：
1. **在线学习有效**：100 episodes 的在线训练使虚拟行人场景成功率从 0.52 → 0.82，物理行人从 0.20 → 0.60
2. **执行时间增加**：学习后机器人更倾向等待行人通过而非强行穿插，这是更安全的社交行为
3. **物理行人更难**：非合作行人不会主动避让机器人，碰撞率更高
4. **学到了避碰行为**：Figure 7/8 对比显示，学习前机器人会强行穿插导致碰撞，学习后会等待行人通过

## 五、方法优势与局限性分析

### 5.1 核心优势

| 优势 | 说明 |
|------|------|
| **无需 Replay Buffer** | 内存占用极低，适合边缘设备（Jetson AGX Orin） |
| **真实世界在线学习** | 无需仿真-真实迁移，直接在物理环境中适应 |
| **安全保底** | SFM 基础策略确保训练初期也不会完全随机行动 |
| **训练稳定** | 三项稳定化技术（归一化+TD缩放+温度调节）避免灾难性失败 |
| **架构简洁** | 不使用 Target Network、Double Q、大型缓冲区 |
| **性能接近 SOTA** | 无 Buffer 却接近有 Buffer 的 SAC/TD3 的性能 |

### 5.2 局限性

| 局限 | 说明 |
|------|------|
| **2D 导航** | 仅处理平面导航，不涉及3D空间或楼梯等场景 |
| **纯 LiDAR** | 不使用视觉信息，无法识别行人意图、姿态、表情 |
| **仅避碰** | 没有更高级的社交规范（如右侧通行、让路给老人等） |
| **物理实验规模小** | 仅 5 个物理行人测试 trial，统计显著性不足 |
| **行人检测依赖 DR-SPAAM** | 需要额外过滤静态误检，鲁棒性有限 |
| **无长期记忆** | 每次部署需要重新在线学习，不保留跨 session 的经验 |
| **奖励函数简单** | 仅有碰撞惩罚和目标接近奖励，缺少社交距离等细粒度奖励 |
| **真实世界成功率仍不高** | 物理行人场景学习后成功率仅 0.60，碰撞率仍有 0.40 |

### 5.3 与 Kinbot 的潜在关联

| 维度 | IRRL | Kinbot 可参考点 |
|------|------|---------------|
| 硬件平台 | Jetson AGX Orin + ROS 2 | Kinbot 同样使用 ROS 2 + 边缘计算 |
| 在线学习 | 真实世界增量 RL | Kinbot P1-low 阶段可引入类似的在线微调 |
| 残差 RL | SFM + 残差修正 | Kinbot 可用 DWA/TEB 作为基础策略 + 残差修正 |
| 行人检测 | LiDAR + DR-SPAAM | Kinbot 有视觉系统，可用更丰富的行人特征 |
| 社交规范 | 仅避碰 | Kinbot 可在奖励函数中加入更多社交约束 |

## 六、总结

IRRL 论文的核心价值在于**证明了一个概念**：在边缘设备上、不使用 Replay Buffer、仅通过单步增量更新，就可以让机器人在真实世界中学会基本的社交导航行为。虽然最终性能（物理行人成功率 0.60）距离实用仍有距离，但这条"轻量化真实世界 RL"的技术路线本身具有重要意义——特别是对于需要在部署后持续适应新环境的机器人产品（如 Kinbot）。

**一句话总结**：SFM 保底 + 轻量残差学习 + 单步增量更新 = 边缘设备上的真实世界社交导航在线学习。
