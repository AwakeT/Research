# 银河通用（Galbot / GalaxyBot）公开资料梳理

> 说明：以下内容按“可验证公开事实”与“更偏公司/媒体宣传口径”区分。判断标准大致为：
> - **较可靠公开事实**：官网静态信息、可核验论文/arXiv/项目页、可回溯的官方新闻页、可直接核验的产品参数。
> - **待独立验证**：明显带有“全球首个”“广泛应用”“率先落地”“产品级验证”等宣传色彩，或主要来自公司自述、媒体转述但缺少第三方经营/客户/部署证据。

## 公开事实

### 1. 公司与组织层面

- 银河通用官网“关于我们”页面公开写明：**北京银河通用机器人股份有限公司**成立于 **2023 年 5 月**，在**北京、深圳、苏州、香港**设有研发中心。官网还写明与**北京大学、北京智源人工智能研究院、宣武医院、北京中关村学院**成立了联合实验室/研究中心。该信息来自官网前端静态内容，可视为公开可核验事实。  
- 官网隐私政策公开披露公司主体为**北京银河通用机器人股份有限公司**，地址为**北京市海淀区海淀大街 3 号 1 幢 14 层 1401**，隐私政策更新时间为 **2026-01-25**。这类法务页信息通常可信度较高。  
- 官网“发展历程”静态内容显示：**2023 年 5 月公司成立**，**2024 年 6 月完成数亿元新一轮融资**。其中“完成融资”若仅以官网单方陈述为依据，属于公开信息，但融资金额与轮次细节仍宜结合工商/媒体融资报道复核。  

### 2. 公开技术路线：可直接核验到的部分

#### 2.1 基础研究与前序能力（官网“技术研究”）
官网“技术研究”页公开列出银河通用/相关团队已对外展示的若干技术方向：
- **材质无关的泛化物体抓取技术**；
- **Open6DOR**：官网称其为“开放指令、可控制物体朝向的物体取放大模型系统”；
- **NaVid**：官网称其为“场景泛化的具身导航大模型”；
- **双臂协同柔性操作**（例如挂衣服）；
- **DexGraspNet**：官网提到其为百万级灵巧手数据集，并提及 ICRA 2023 最佳操作论文提名。  

这些条目说明银河通用公开叙事中的核心路线，并不是从 2025 年才突然出现，而是沿着**抓取/操作、导航、数据集与基础模型**逐步推进。

#### 2.2 GraspVLA：抓取方向的 VLA 基础模型
这是目前最容易核验、证据最强的技术路线之一。

- arXiv 论文 **GraspVLA: a Grasping Foundation Model Pre-trained on Billion-scale Synthetic Action Data**（arXiv:2505.03233）可直接核验。论文作者包含 **He Wang（王鹤）** 等人。  
- 论文与项目页明确写到：
  - 构建了 **SynGrasp-1B**，即**十亿帧规模**的仿真抓取数据集；
  - 目标是探索**完全基于大规模合成动作数据**训练 VLA 模型；
  - 模型把**视觉-语言骨干**与**flow-matching 动作生成**结合；
  - 通过同时利用**互联网语义数据**与**合成动作数据**，增强 open-vocabulary 抓取泛化；
  - 在真实与仿真环境都做了评测，并强调**零样本泛化**与**少样本后训练适配**。  
- 项目页还列出了几个明确应用外延：
  - **Industry: new vocabulary**：工业场景中通过少量标注适配特殊部件命名；
  - **Retail: novel action patterns**：零售场景中学习从密集摆放瓶装商品中顺序抓取；
  - **Home: align with human preference**：家庭场景中学习符合人偏好的抓取姿态。  

这说明银河通用公开技术路线至少有一条非常明确：**“合成数据大规模预训练 → 通用抓取基础模型 → 少样本后训练迁移到工业/零售/家庭具体场景”**。

#### 2.3 TrackVLA：导航/跟随方向的 VLA 模型
- arXiv 论文 **TrackVLA: Embodied Visual Tracking in the Wild**（arXiv:2505.23189）可直接核验，作者同样包含 **He Wang**。  
- 论文摘要显示：
  - 任务是**Embodied Visual Tracking（具身视觉跟踪）**，本质上是机器人在动态环境中依靠第一视角视觉持续跟踪目标；
  - TrackVLA 将**目标识别**与**轨迹规划**通过共享 LLM backbone 联合建模；
  - 构建了 **EVT-Bench**，并收集 **170 万样本**；
  - 声称在仿真和真实环境中都取得较强泛化，并在真实环境中达到 **10 FPS** 推理速度。  

这条路线与 GraspVLA 互补：
- **GraspVLA** 解决“看懂+抓到”；
- **TrackVLA** 更接近“看懂+跟住+走到”。

它说明银河通用在公开学术路线里，至少同时推进了：
1. **操作/抓取 VLA**；
2. **导航/跟随 VLA**；
3. **大规模仿真合成数据驱动的具身模型训练**。

#### 2.4 AstraBrain / AstraSynth / DeepVLA / 产品矩阵
- 在本次可直接抓取并核验到的官网公开内容、论文页、新闻页中，**AstraBrain / AstraSynth / DeepVLA** 这些名字**没有找到足够强的原始官方页面证据**。  
- 因此当前更稳妥的写法是：
  - **若外部材料提到 AstraBrain / AstraSynth / DeepVLA，暂不能在本次已核验源中确认其是否为银河通用已正式对外稳定发布的官方技术品牌/系统名**；
  - 在正式结论中应标注为**“待独立验证”**，不要与 GraspVLA / TrackVLA 这种有 arXiv 和项目页支持的内容等量齐观。  
- 就可核验证据看，银河通用现阶段更明确的公开技术主线，仍是：**合成数据生成范式 + 抓取 VLA + 跟踪/导航 VLA + 通用机器人平台化落地**。

### 4. 已公开可验证的零售、工业、导航、重载、常态化运营信息

#### 4.1 零售场景
- **GraspVLA 项目页**明确把“Retail: novel action patterns”作为应用示例：通过少量后训练轨迹，让模型学习在**密集摆放瓶装商品**环境中顺序抓取，并迁移到未见过的瓶子。  
  - 这是**技术可行性演示层面**的可验证事实；
  - 但它**不等于已经大规模商业部署**。  
- IT之家 2025-06-01 报道转述银河通用说法：**TrackVLA 已在超市、商场、服装店等未训练真实场景中完成长程验证**，包括目标切换、儿童提醒、避障等。  
  - 这说明银河通用**公开宣称**其导航/跟随模型做过零售环境长程验证；
  - 但目前缺少商场方、连锁零售客户、采购/部署公告等**独立第三方证据**，因此更适合归类为“已公开演示/验证”，不宜直接写成“已规模化落地”。
- 官网新闻列表出现：**“圆明园机器人店员上岗”**（2025-09-12，来源标签显示“北京海淀”）。  
  - 该标题至少说明银河通用官网曾公开过**机器人店员上岗**相关案例；
  - 但仅凭新闻列表，尚未拿到正文、客户方证词和运营指标，故可记为“公开案例线索”，仍需独立验证细节。  
- IT之家另有报道标题：**“美团买药联合银河通用，展出全球首个人形机器人智慧药房解决方案”**。  
  - 这是一个比较强的**公开合作线索**；
  - 但本次未完整提取到正文，且“展出解决方案”与“常态化运营”不是一回事，因此最多可视为**零售/医药零售方向的联合展示或方案验证证据**，不能直接等同于大规模上线运营。

#### 4.2 工业场景
- **GraspVLA 项目页**明确把“Industry: new vocabulary”作为应用外延：少量带框标注即可让模型识别工业场景中的**稀有部件命名**并完成抓取迁移。  
  - 这可以作为银河通用技术路线中**工业抓取适配能力**的公开事实；
  - 但它仍属于模型能力展示，**不是客户交付清单**。  
- 官网合作页写明其产品能力覆盖**工业制造、仓储物流**等场景。  
  - 这是官网公开口径，可记录为公司自我定位；
  - 但没有发现足够多第三方客户案例、产线名称、班次时长、替代人数、MTBF 等硬指标，因此工业常态化运营证据仍偏弱。

#### 4.3 导航/跟随/VLN 相关场景
- **TrackVLA 论文**本身就是极强证据：说明银河通用/相关团队在**具身视觉跟踪、目标识别、轨迹规划联合建模**上已有明确研究产出。  
- IT之家 2025-06-01 报道与公司公开视频口径指向：TrackVLA 被描述为**产品级导航大模型**，特点是：
  - **纯视觉环境感知**；
  - **自然语言指令驱动**；
  - **不依赖预建图**；
  - 在购物中心、电梯、游乐区等复杂环境做长程跟随；
  - 联合**宇树机器狗**做过验证。  
- 其中“论文里确有 TrackVLA、真实环境评测、10 FPS”是公开事实；
- 但“产品级 FSD、长程严苛验证已经满足产品使用要求”等更强结论，还需要更多第三方验证。

#### 4.4 重载信息
- 在本次已核验到的公开材料中，**没有找到明确、可核验的“重载搬运/重载操作”性能数据**，例如额定负载、臂展负载曲线、整机搬运公斤数、托盘/料箱处理等。  
- 因此“重载”这一项目前**缺乏硬证据支持**，应明确标注：**待独立验证**。  

#### 4.5 常态化运营信息
- 官网“关于我们”中使用了“已率先在商业、工业、医疗等场景中广为应用”的表述。  
- 但截至本次可核验资料：
  - 尚未找到稳定的**客户名单**；
  - 尚未找到按天/按班次/按门店/按站点的**持续运营指标**；
  - 尚未找到第三方公开披露的**采购规模、续约情况、上线时长、出勤率、故障率**。  
- 因此，关于“常态化运营”的最稳妥判断是：
  - **有若干公开演示、联合展示、真实场景验证线索**；
  - **但是否已形成大规模、长期、稳定、可复用的常态化商业运营，公开证据仍不足**。

## 公开宣传口径

以下内容多来自公司官网、官网新闻标题或媒体转述，具有较强宣传色彩；如要在正式研究结论中使用，建议统一标注 **“待独立验证”**。

### 1. 技术定位类宣传口径
- “**市场领先**的具身多模态大模型通用机器人企业”；
- “**引领行业前沿**”；
- “**全球首个**全面泛化端到端具身抓取基础大模型 GraspVLA”；
- “**全球首个产品级端到端具身 FSD 大模型** TrackVLA”；
- “世界第一个开放指令且可控朝向的 Open6DOR”；
- “世界第一个场景泛化的具身导航大模型 NaVid”。

这些说法可能部分有技术依据，但“全球首个/世界第一/市场领先”本身需要系统性竞品对比与时间线对照，**不能仅凭公司单方表述直接下结论**。

### 2. 商业化与规模化类宣传口径
- “已率先在**商业、工业、医疗等场景中广为应用**”；
- “形成**完善的人形机器人产品矩阵**”；
- “覆盖工业、仓储物流、智慧服务、科研教育、康养理疗等核心场景”；
- “圆明园机器人店员上岗”若被进一步解读为“已成熟常态化运营”，则也需要谨慎。  

这些表述可以作为公司市场定位与销售话术的参考，但如果用于投资/竞对报告，应进一步补充：**具体客户、部署点位、运行周期、复购与扩点情况**。

### 3. 导航/看护/守护类更强主张
IT之家转述的 TrackVLA 八大能力里，以下几类尤其需要独立验证：
- 可长期稳定不跟错人；
- 丢失目标后可稳定找回；
- 在陌生商场、电梯、游乐区无需建图直接部署；
- App 远程守护、主动提醒儿童奔跑/老人跌倒；
- 能泛化跟随宠物，甚至偶遇的动物狗；
- 在未训练真实场景完成“严格长程验证”。

这些叙述很像产品发布会上的亮点总结，**不代表没有可能实现**，但目前公开可回溯证据更偏视频演示与媒体转述，而非系统测试报告，因此应标为 **待独立验证**。

### 4. AstraBrain / AstraSynth / DeepVLA
- 这些名称在本次可核验原始材料里**未形成足够闭环证据**；
- 如果其他二手资料将其描述为银河通用核心技术栈，应统一标记 **待独立验证**；
- 在没有官方原始页面、论文或产品手册支撑前，不建议将其写成已确认事实。

## 对 VLN/VLA 的启示

### 1. 银河通用的路线更像“按技能拆解的具身模型栈”
从可核验材料看，银河通用并不是一开始就追求一个包打天下的单一通用体模型，而是至少沿着几条可落地技能链推进：
- **GraspVLA**：聚焦抓取/操作；
- **TrackVLA**：聚焦跟踪/导航；
- 更早的 **NaVid / Open6DOR / DexGraspNet** 等，像是为后续 VLA 体系准备任务、数据与先验。  

这对 VLN/VLA 的启示是：**通用具身智能的商业化落地，短期内可能更依赖“按高价值技能纵深突破”，而不是一步到位的大一统模型。**

### 2. 合成数据不是辅助，而是主训练范式
GraspVLA 的核心亮点，不只是“用了仿真”，而是明确提出：
- **用十亿帧级别合成动作数据做预训练主干**；
- 再通过互联网语义和少量真实后训练去补齐语义与偏好。  

这对 VLN/VLA 很关键：
- 导航、跟随、抓取等任务如果要覆盖大量**长尾环境与交互变化**，真实采集成本极高；
- 银河通用路线说明：**高质量合成数据 + domain randomization + 任务结构设计**，可能是把 VLA 从 demo 推向产品化的现实路径。  

### 3. VLA 的商业价值不只在“会动”，而在“后训练成本足够低”
无论是 GraspVLA 的 few-shot post-training，还是 TrackVLA 的“陌生环境可部署”叙事，核心商业含义都是同一件事：
- **基础模型先学通用能力**；
- 进入行业时，靠**极少量数据/规则/示教**适配具体客户。  

对 VLN/VLA 来说，这意味着真正决定商业化速度的，不只是 benchmark 分数，而是：
- 新环境冷启动成本；
- 对新目标、新货品、新路线的适配样本量；
- 是否能把“部署工程”压缩为“轻量后训练 + 安全规则层”。

### 4. VLN/VLA 落地最缺的不是 demo，而是运营证据
银河通用公开资料的一个明显特征是：
- **技术论文与演示很多**；
- **长期运营指标很少**。  

这对行业判断是个提醒：
- 一个具身/VLA 公司是否真的进入“商业拐点”，不能只看模型名字和视频；
- 更要看：**连续运行时长、故障率、人工接管率、部署密度、客户复购、单位经济性**。  

换句话说，VLN/VLA 从研究走向产业，最后拼的不是“会不会”，而是“能不能稳定地、便宜地、持续地干活”。

### 5. 对做 VLN 的团队尤其有价值的一点：导航应与识别/交互联合建模
TrackVLA 的关键点不只是 tracking，而是把**目标识别**与**轨迹规划**在 VLA 内统一。对 VLN 任务而言，这意味着：
- 传统“感知模块 + 规划模块”拆分在动态场景里很容易断链；
- 将**语言目标、视觉目标识别、短期轨迹生成、闭环纠错**放进同一模型范式，可能更适合真实商场/门店/医院这类动态环境；
- 未来面向服务机器人、陪护机器人、巡检机器人的 VLN，可能会越来越像**TrackVLA 这种识别-跟踪-导航一体化结构**。
